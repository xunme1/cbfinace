# 旧版信号与新版信号说明

本文说明当前项目中两套信号体系的差异、计算方式和适用场景。

## 总体区别

| 项目 | 旧版信号 | 新版信号 |
| --- | --- | --- |
| 核心视角 | 机构阵营 vs 散户阵营 | 五大席位逐一追踪 |
| 主要页面 | `/signals`、旧 Dashboard | `/seat-tracker`、`/products/:product` |
| 核心问题 | 机构和散户是否对手盘、共振或单边行动 | 哪些品种五大席位偏多、偏空或对峙 |
| 聚合方式 | 先把席位归为机构 / 散户阵营 | 保留五大席位各自变化 |
| 展示重点 | 信号类型、机构净变化、散户净变化 | 品种、板块、主要变化合约、五大席位净变化 |
| 是否使用强度 | 旧后端仍保留 `strength` 字段 | 新接口不返回 `strength` |
| 是否使用交易所 | 不使用 | 不使用 |

## 旧版信号

旧版信号基于“机构阵营”和“散户阵营”的整体净变化判断。

### 阵营划分

机构阵营：

```text
高盛期货
摩根大通
国泰君安
```

散户阵营：

```text
东方财富
徽商期货
```

其他席位标记为“其他”，不作为机构 / 散户核心信号判断。

### 基础公式

每条持仓记录先计算：

```text
net_change = long_change - short_change
```

然后按品种和阵营聚合：

```text
institution_net_change = 机构阵营 net_change 之和
retail_net_change = 散户阵营 net_change 之和
```

方向判断使用噪音阈值：

```text
NOISE = 500
```

规则：

```text
net_change > 500   => 做多
net_change < -500  => 做空
其他               => 中性
```

### 旧版信号类型

| 信号类型 | 触发条件 | 含义 |
| --- | --- | --- |
| 对手盘 | 机构做多、散户做空；或机构做空、散户做多 | 两个阵营方向相反 |
| 共振 | 机构和散户同时做多；或同时做空 | 两个阵营方向一致 |
| 机构突击 | 机构明显行动，散户中性 | 机构单边变化更明显 |
| 散户自嗨 | 散户明显行动，机构中性 | 散户单边变化更明显 |
| 噪音 | 双方变化都不明显 | 暂无明显方向 |

### 旧版信号适用场景

适合快速回答：

```text
机构和散户是不是在反向交易？
某个品种是否出现机构单边动作？
某个品种是否出现双方共振？
```

### 旧版信号局限

- 阵营划分较粗，会掩盖单个席位之间的分歧。
- 不同品种持仓规模差异大，旧版绝对变化值不适合跨品种直接比较。
- 旧版 `strength` 只是绝对净变化合计，不应理解为交易建议。
- 品种详情对合约和席位层面的解释不够细。

## 新版信号

新版信号基于五大席位逐一分析，不再把席位粗略合并为机构 / 散户阵营。

### 跟踪席位

第一阶段固定跟踪五大席位：

```text
高盛期货
摩根大通
国泰君安
东方财富
徽商期货
```

如果某个品种缺少某个席位的数据，该席位按 0 处理，方向为“中性”。

### 基础公式

按 `product + broker` 聚合：

```text
broker_long_change = sum(long_change)
broker_short_change = sum(short_change)
broker_net_change = broker_long_change - broker_short_change
```

按 `product + contract` 聚合：

```text
contract_long_change = sum(long_change)
contract_short_change = sum(short_change)
contract_net_change = contract_long_change - contract_short_change
```

方向判断：

```text
net_change > 500   => 偏多
net_change < -500  => 偏空
其他               => 中性
```

### 主要变化合约

第一阶段的“主要变化合约”不是交易所官方主力合约，而是按变化量识别：

```text
按 abs(contract_net_change) 从大到小排序
第一名：主要变化合约
第二名：次要变化合约
```

这样可以回答：

```text
这个品种今天的变化主要集中在哪个合约？
```

### 新版信号类型

新版信号返回：

```text
strong_long
strong_short
conflict
positive
reverse
neutral
```

对应中文：

```text
强看多
强看空
冲突
正向
反向
中性
```

### 新版信号规则

先统计五大席位方向：

```text
long_count = 偏多席位数
short_count = 偏空席位数
neutral_count = 中性席位数
```

规则：

| 信号 | 条件 | 含义 |
| --- | --- | --- |
| 强看多 | `long_count >= 3` 且 `short_count <= 1` | 五大席位多数偏多 |
| 强看空 | `short_count >= 3` 且 `long_count <= 1` | 五大席位多数偏空 |
| 冲突 | `long_count >= 2` 且 `short_count >= 2` | 多空席位明显对峙 |
| 正向 | 主要变化合约方向与全部合约聚合方向一致，且不为中性 | 主变化合约和整体方向一致 |
| 反向 | 主要变化合约方向与全部合约聚合方向相反 | 主变化合约和整体方向背离 |
| 中性 | 其他情况 | 暂无明确结构 |

### 对峙判断

品种详情页会单独输出对峙说明。

判断规则：

```text
偏多席位数 >= 2 且 偏空席位数 >= 2
```

示例：

```text
国泰君安、摩根大通偏多；东方财富、徽商期货偏空，主力席位之间存在明显对峙。
```

如果不满足对峙条件：

```text
五大席位整体方向较一致，暂未出现明显对峙。
```

### 新版信号适用场景

适合回答：

```text
今天哪些品种五大席位明显偏多？
今天哪些品种五大席位明显偏空？
哪些品种主力席位之间存在冲突？
某个品种的变化主要集中在哪个合约？
某个品种是哪些席位在推动？
```

## 为什么新版不展示 strength

新版第一阶段不展示 `strength`，原因是：

```text
1. 不同品种持仓规模差异很大。
2. 绝对变化值不能直接跨品种比较。
3. strength 容易被误解为交易强弱或交易建议。
4. 当前目标是追踪席位结构，而不是做品种排名打分。
```

如果需要排序，新版页面默认按：

```text
abs(all_contract_net_change)
```

排序，但前端只称为“全部合约聚合净变化”，不称为强度。

## 使用建议

### 看全市场结构

优先看：

```text
/seat-tracker
```

关注：

```text
强看多
强看空
冲突
主要变化合约
五大席位净变化
```

### 看单品种细节

进入：

```text
/products/:product?date=YYYY-MM-DD
```

关注：

```text
五大席位总体变化
合约汇总
合约-席位明细
席位对峙分析
```

### 做历史对照

可以保留使用旧版：

```text
/signals
```

但旧版信号更适合做“机构 / 散户阵营关系”观察，不适合作为新版席位追踪的替代。

## 注意事项

- 当前 CSV 没有 `exchange` 字段，因此不要实现或使用交易所筛选。
- 当前信号只是结构化观察，不代表交易建议。
- 新版“主要变化合约”按变化量识别，不等同于交易所官方主力合约。
- 后续如接入资金流向数据，可以再扩展“资金流入前 5 品种”和资金维度的对峙分析。

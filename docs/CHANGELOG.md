# 更新日志

## 2026-06-12：新增历史趋势模块

### 新增

- 新增历史趋势页面：`/trends`。
- 新增品种历史趋势详情页：`/trends/:product`。
- 新增后端接口：`GET /api/trends`。
- 新增后端接口：`GET /api/trends/products/{product}`。
- 支持按结束日期、交易日范围、板块、最新信号和品种关键词筛选。
- 支持查看最新信号、信号连续天数、主力/散户最新净变化、主力/散户累计净变化和趋势状态。
- 单品种趋势详情支持查看信号时间线、主力 vs 散户净变化趋势、五大席位净变化趋势、主要合约净变化趋势和每日明细。

### 说明

- 第一版历史趋势基于本地已有 `positions_日期.csv` 文件计算。
- 日期范围按已存在的持仓 CSV 文件倒序选取，不额外触发抓取。

## 2026-06-12：新增席位对对碰

### 新增

- 新增“席位对对碰”页面：`/seat-battle`。
- 新增近一月数据补抓脚本：`backend/scripts/backfill_market_data.py`。
- 支持用户从五大席位中自由选择阵营A和阵营B，每个阵营可选择一个或多个席位。
- 支持按日期、板块、品种关键词和对比信号筛选。
- 输出两个阵营在各品种上的持仓净变化、方向、差值和合计变化。
- 支持点击品种进入席位对对碰详情页：`/seat-battle/products/:product`。
- 详情页支持查看单个品种下各合约的阵营对比，以及精确到“合约-席位”的持仓明细。

### 说明

- 默认阵营A为高盛期货、摩根大通、国泰君安。
- 默认阵营B为东方财富、徽商期货。
- 两个阵营不能选择相同席位。
- 批量补抓脚本默认只抓工作日，并跳过已有 CSV。

## 2026-06-11：优化主力 / 散户反向信号定义

### 调整

- 新版强信号不再按“五大席位多数同向”判断。
- `强看多` 改为：三大主力席位整体偏多，同时两大散户席位整体偏空。
- `强看空` 改为：三大主力席位整体偏空，同时两大散户席位整体偏多。
- 主力席位与散户席位同向时，只归为“正向 / 同向结构”，不再作为强信号。
- 品种详情页的对峙说明改为强调“主力席位 vs 散户席位”的反向结构。

### 业务口径

- 三大主力席位：高盛期货、摩根大通、国泰君安。
- 两大散户席位：东方财富、徽商期货。
- 强信号来自主力与散户反向，而不是所有席位同向。
- 机构和散户同时做多或同时做空，只说明方向一致，缺少散户反指带来的反向确认。

  - `GET /api/seat-tracker/categories`
  - `GET /api/products/{product}/dashboard`
- 新增五大席位追踪逻辑：
  - 高盛期货
  - 摩根大通
  - 国泰君安
  - 东方财富
  - 徽商期货
- 新增按板块筛选、信号筛选、品种关键词搜索。

- 新增席位对峙分析说明。
- 新增 `project.md`，记录本阶段重构需求和后续规划。

### 调整

- 系统标题调整为“期货席位追踪系统”。
- 首页默认分析日期调整为 `2026-06-09`。
- 首页和旧版信号页不再突出展示“信号强度”。
- `/products/:product` 改为新版品种席位持仓详情页。
- 导航新增“席位追踪”，原信号列表改名为“旧版信号”。
